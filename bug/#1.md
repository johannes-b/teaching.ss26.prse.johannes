### Bug Report

**Title:** Calculator display initialises with `0` instead of an empty field

---

### Description

When the calculator application starts, the display immediately shows `0`.
The display should be blank on startup so the user can enter the first operand
without having to clear a pre-filled value.

---

### Steps to Reproduce

1. Launch the application (`mvn javafx:run`).
2. Observe the display field without pressing any button.

**Expected:** Display is empty.
**Actual:** Display shows `0`.

---

### Root Cause

The `TextField` in `src/main/resources/at/jku/se/calculator/calculator.fxml` (line 15)
is declared with a hard-coded `text="0"` attribute:

```xml
<TextField fx:id="display" text="0" ... />
```

---

### Acceptance Criteria

- [ ] On startup the display is empty (no `0` pre-filled).
- [ ] Pressing `CE` still resets the display to `0` (existing clear behaviour is unchanged).
- [ ] All existing tests continue to pass.

---

### Related

- `calculator.fxml` — line 15, `text="0"` attribute
- `CalculatorController.java` — `clear()` method returns `"0"` (CE button, keep as-is)
